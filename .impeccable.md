# Project: Bytes Marketplace

Mobile-friendly marketplace web app (React + Vite, Nx monorepo, Bun), packaged to native via Capacitor. Stack: TypeScript (no `any`), MUI v9 with the styled-components engine, Redux Toolkit, Redux Saga. Frontend-only; all backend data is mocked via delayed promises. Built to scale to thousands of products (pagination, search, category filter, sort). Designed mobile-first — touch targets, responsive layouts, safe-area aware.

## Design Context

### Users
Shoppers browsing a general-goods catalog (electronics, beauty, home, groceries — DummyJSON-style) on their phone. Context: casual browsing across daylight and evening sessions. Job to be done: scan many products fast, drill into details, build a cart, and place an order without friction. Frustration-free is the explicit bar — every flow needs loading, empty, and error states.

### Brand Personality
Editorial, confident, considered. Three words: **curated, calm, precise.** The interface should feel like a well-designed retail magazine, not a discount aggregator. Evoke trust and quiet quality over urgency or hype.

### Aesthetic Direction — "Mono Editorial"
Ported from the approved Claude Design prototype ("Marketplace app design task"). Monochrome, editorial, structural. **No accent colour** — the design draws all its energy from contrast, weight, and hard structure, not hue:
- **Squared, never rounded.** Borders are hard `1.5px` solid `ink`. No drop shadows on content (only the device/page surface).
- **Two typefaces, strict roles.** `Archivo` for everything human (headings 800 weight, tight `-0.02/-0.03em` tracking; body 500–600). `Spline Sans Mono` for every machine/label string — prices, ratings, SKUs, category chips, counts, button labels — usually UPPERCASE with `.04–.14em` letter-spacing.
- **Ink-on-bg buttons.** Primary actions are solid `ink` background, `bg` foreground, square. Secondary actions are bordered/ghost.
- **Striped tiles for imagery.** Product images are diagonal `repeating-linear-gradient` stripes over `tile` (placeholder for real photos).
- **Both light and dark**, swapped via the token set below. Anti-references unchanged (no neon, gradient text, side-stripe cards, glassmorphism).

### Design Principles
1. **Structure is the style.** Hierarchy from borders, weight, and mono/sans contrast — not colour or decoration.
2. **Mono labels, sans content.** If it's a number, code, or label → Spline Sans Mono. If it's a name or sentence → Archivo.
3. **Every async flow has all four states.** Loading, empty (teaches next step), error (recoverable, e.g. "No charge was made — try again"), success. No dead ends.
4. **One ink, used everywhere structural.** Borders, primary buttons, active states all share `--ink`. The contrast IS the brand.
5. **Fast-feeling.** Optimistic cart updates, snappy transitions (transform/opacity only — `mslide`/`mpop` 0.2–0.3s ease).
